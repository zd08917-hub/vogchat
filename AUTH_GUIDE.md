# Руководство по использованию улучшенной системы аутентификации

## Для пользователей

### Вход в аккаунт

Теперь у вас есть два способа входа:

#### 1. Вход по email-коду (традиционный способ)
- Введите ваш email
- Нажмите "Продолжить"
- Проверьте почту и введите 6-значный код
- Нажмите "Войти"

#### 2. Вход по паролю (новый способ)
- Введите ваш email
- Выберите "Пароль" в переключателе методов
- Введите ваш пароль
- Нажмите "Войти по паролю"

### Установка пароля

Если у вас еще нет пароля, вы можете его установить:

1. **После регистрации** - система предложит установить пароль
2. **Через настройки безопасности** - в разделе "Безопасность" → "Пароль аккаунта"
3. **При первом входе по email-коду** - система предложит установить пароль для удобства

**Требования к паролю:**
- Минимум 8 символов
- Должен содержать буквы и цифры
- Рекомендуется использовать уникальный пароль

### Управление методами входа

Вы можете выбрать предпочтительный способ входа:

1. Откройте **Настройки** (шестеренка в интерфейсе)
2. Перейдите в раздел **Безопасность**
3. В разделе **"Метод входа"** выберите:
   - **Email код** - каждый раз получать код на почту
   - **Пароль** - быстрый вход по паролю

### Если забыли пароль

1. На странице входа выберите метод "Пароль"
2. Нажмите **"Забыли пароль?"**
3. Введите ваш email
4. Проверьте почту и следуйте инструкциям в письме
5. Установите новый пароль

## Для администраторов/разработчиков

### Настройка отправки email

#### Режим разработки (по умолчанию)
- Коды отправляются в консоль сервера
- Не требуется настройка SMTP
- Для тестирования функционала

#### Режим продакшена
1. Настройте SMTP сервер в файле `server/.env`:
```env
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your_email@gmail.com
SMTP_PASS=your_app_password
NODE_ENV=production
```

2. Перезапустите сервер

### Миграция базы данных

При первом запуске после обновления:
```bash
cd server
npm run migrate
```

Это добавит необходимые поля для работы с паролями.

### Тестирование системы

#### Тестовые сценарии:

1. **Новый пользователь:**
   - Регистрация → Вход по email-коду → Установка пароля → Вход по паролю

2. **Существующий пользователь:**
   - Вход по email-коду → Установка пароля → Переключение на вход по паролю

3. **Сброс пароля:**
   - Запрос сброса → Получение кода на email → Установка нового пароля

4. **Мультиустройственность:**
   - Вход на разных устройствах с разными методами

#### Команды для тестирования API:
```bash
# Проверка наличия пароля у пользователя
curl "http://localhost:3001/api/auth/has-password?userId=USER_ID"

# Установка пароля
curl -X POST http://localhost:3001/api/auth/set-password \
  -H "Content-Type: application/json" \
  -d '{"userId": "USER_ID", "password": "Test1234"}'

# Вход по паролю
curl -X POST http://localhost:3001/api/auth/login-password \
  -H "Content-Type: application/json" \
  -d '{"email": "user@example.com", "password": "Test1234"}'
```

## Часто задаваемые вопросы

### В: Почему я не могу выбрать вход по паролю?
**О:** У вас должен быть установлен пароль. Сначала войдите по email-коду и установите пароль в настройках безопасности.

### В: Что делать, если я не получаю email с кодом?
**О:**
1. Проверьте папку "Спам"
2. Убедитесь, что email введен правильно
3. В режиме разработки проверьте консоль сервера
4. В режиме продакшена проверьте настройки SMTP

### В: Можно ли использовать оба метода входа?
**О:** Да, вы можете использовать любой метод независимо от предпочтительного. Предпочтительный метод - это просто настройка по умолчанию.

### В: Безопасно ли хранить пароли?
**О:** Да, пароли хэшируются с использованием bcrypt, что является промышленным стандартом. Исходные пароли никогда не хранятся в базе данных.

### В: Что происходит с существующими пользователями?
**О:** Все существующие пользователи продолжают работать через email-коды. Они могут установить пароль в любое время.

## Устранение неполадок

### Проблема: Ошибка "Неверный пароль"
**Решение:**
1. Проверьте правильность ввода пароля
2. Используйте функцию "Забыли пароль?" для сброса
3. Убедитесь, что Caps Lock выключен

### Проблема: Ошибка "У пользователя не установлен пароль"
**Решение:** Войдите по email-коду и установите пароль в настройках безопасности.

### Проблема: Email не отправляется
**Решение:**
1. **В режиме разработки:** проверьте консоль сервера
2. **В режиме продакшена:**
   - Проверьте настройки SMTP в `.env`
   - Убедитесь, что почтовый сервер доступен
   - Проверьте логи сервера на наличие ошибок

### Проблема: Не применяются миграции
**Решение:**
```bash
# Откат миграций
cd server
npm run migrate:rollback

# Повторное применение
npm run migrate
```

## Контакты для поддержки

При возникновении проблем:
1. Проверьте логи сервера в консоли
2. Убедитесь, что база данных доступна
3. Проверьте настройки окружения
4. Обратитесь к разработчикам с описанием проблемы
